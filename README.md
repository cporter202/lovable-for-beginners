## Build more than a prototype

Lovable can take an idea from a natural-language prompt to a deployed full-stack web application. This course teaches the entire workflow, including the parts short tutorials usually skip: planning, data ownership, authorization, testing, security reviews, release discipline, analytics, monitoring, Git collaboration, and hosting choices.

> [!TIP]
> **Learn by building.** Keep one practice project open as you work through the modules. Each lesson includes prompts, checkpoints, acceptance criteria, and direct links back into Lovable.

| At a glance | |
| --- | --- |
| **15 focused modules** | Foundations through production operations |
| **Hands-on capstone** | Build a secure multi-user Book Club Hub |
| **Beginner accessible** | No coding experience required for Modules 1-6 |
| **Production minded** | RLS, secrets, tests, scans, deployment, and monitoring |
| **Documentation reviewed** | Audited against official Lovable docs on July 18, 2026 |

## Why this course

| You will learn to | What that means in practice |
| --- | --- |
| **Direct Lovable clearly** | Use Plan and Build modes, strong prompts, knowledge, references, skills, and subagents |
| **Build complete products** | Add database, authentication, storage, server functions, integrations, AI, and payments |
| **Protect users and data** | Enforce authorization with RLS, keep secrets server-side, and run security scans |
| **Prove the app works** | Use browser testing, frontend tests, direct edge calls, edge tests, and debugging evidence |
| **Ship with confidence** | Publish snapshots, connect domains, improve SEO/AEO, review analytics, and monitor releases |
| **Own the code** | Use GitHub or GitLab sync, branches, pull requests, CI, and external hosting when needed |

## Who it is for

- First-time app builders who want a guided path
- Founders turning an idea into a usable product
- Designers, marketers, educators, and operations teams
- Developers collaborating with non-developers
- Teams evaluating Lovable for real production work

No coding experience is required to begin. The later modules introduce code inspection, APIs, Git, and infrastructure in plain language.

## The capstone

In [Module 9](module-09-real-world-project.md), you build **Book Club Hub**, a complete multi-user application with:

- Email and Google authentication
- Private clubs, roles, invitations, books, progress, and notes
- Lovable Cloud database, storage, edge functions, and logs
- Row-level security tested across multiple user roles
- Frontend, backend, and browser verification
- Basic and Deep security scans
- Production publishing, SEO/AEO, analytics, and monitoring

The project is intentionally milestone-based so you learn how to review architecture, data policies, and behavior before piling on more features.

## Quick start

1. **[Open Lovable](https://afflat3a2.com/trk/lnk/7BB81506-2890-47A0-9BDD-D03343EC49CB/?o=32337&c=918277&a=184866&k=D5D811C96B2D90FAF2ABF3287B46C45F&l=38178&s1=github)** and create a free account or enter your workspace.
2. Start with [Module 1](module-01-welcome-to-lovable.md) and create the reading-list practice project.
3. Keep the [Quick Reference](supplement-quick-reference.md) open while you build.
4. Complete Modules 1-10 in order, then choose advanced modules based on your project.
5. Use a disposable project for migrations, destructive experiments, and security exercises.

> [!IMPORTANT]
> Lovable changes quickly. Product controls can move before screenshots or third-party tutorials catch up. This course prioritizes current concepts and links directly to the official documentation for time-sensitive details.
